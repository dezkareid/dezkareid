# Plan: Personal Website for Frontend Developer

## Architecture Overview
- **Framework**: Astro (SSG) for high performance and excellent SEO.
- **Styling**: Vanilla CSS utilizing `@dezkareid/design-tokens`. We will use CSS variables generated by the design system for consistent coloring, typography, and spacing.
- **Content Management**: Astro Content Collections using Markdown (`.md`) for Projects and Services.
- **Component Strategy**: Functional Astro components for the UI, keeping the client-side JS to a minimum.
- **Responsive Strategy**: Mobile-first design using CSS Flexbox and Grid.

## Implementation Phases

### Phase 1: Foundation & Styling
- [ ] Add `@dezkareid/design-tokens` as a workspace dependency (`*`) in `apps/main-website/package.json`.
- [ ] Ensure design tokens are built and available for the workspace.
- [ ] Define global styles in `src/styles/global.css`, importing the design token variables.
- [ ] Create a comprehensive `Layout.astro` with metadata (SEO), favicon integration, and basic semantic structure (Header, Main, Footer).

### Phase 2: Core UI Components
- [ ] Develop basic UI components:
    - `Button`: Using design tokens for variants.
    - `Section`: Container for consistent vertical spacing.
    - `Card`: Base card for projects and services.
- [ ] Implement specific sections:
    - `Hero`: Name, role, and CTA.
    - `SkillsList`: Visual list of technical proficiencies.
    - `SocialLinks`: Icons/links to professional profiles.

### Phase 3: Content & Collections
- [ ] Configure `src/content/config.ts` for `projects` and `services`.
- [ ] Create sample markdown files for at least 3 projects and 3 services.
- [ ] Implement `ProjectCard.astro` and `ServiceCard.astro` to display collection data.

### Phase 4: Pages & Routing
- [ ] **Home Page (`index.astro`)**: Compose sections into a cohesive landing page.
- [ ] **Projects Page (`projects/index.astro`)**: Full list of projects with filtering/categories if needed.
- [ ] **Services Page (`services/index.astro`)**: Detailed overview of professional offerings.
- [ ] **About Page (`about.astro`)**: Professional bio and detailed background.

### Phase 5: Optimization & Launch
- [ ] Add meta tags and Open Graph data for social sharing.
- [ ] Audit for Accessibility (aria-labels, contrast, keyboard navigation).
- [ ] Performance optimization (Image component, minimal scripts).

## Technical Dependencies
- `@dezkareid/design-tokens`: For unified styling variables.
- `astro`: Core framework.
- `@astrojs/check`: For type-checking Astro components.

## Risks & Mitigations
- **Risk**: Missing specific tokens for complex layouts.
  - **Mitigation**: Use local CSS variables for one-off styles; include an annotation or comment to consider if these can be promoted to `@dezkareid/design-tokens` if generally useful.
- **Risk**: Images in projects might slow down initial load.
  - **Mitigation**: Use `<Image />` component from `astro:assets` and provide optimized placeholders.

## Out of Scope
- Interactive blog or comment system.
- Backend database or API integrations.
- User accounts or authentication.
- Real-time chat or contact form backend (will use `mailto:` or external service links).
